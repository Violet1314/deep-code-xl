## v0.38.0（2026-06-28）

- **Edit `no_match` 智能引导** — `old_string` >15 行或 >500 字符时，`no_match` 错误自动建议改用 `start_line`+`end_line` 行号模式，避免 AI 陷入"重读→重试→失败"死循环
- **内置 Skill 规则体系** — `tool-planning` 新增 Edit 工具规则（大段用行号、no_match 后禁止重读、信任 content_hash）与 Read 反模式清单；`cost-control` 新增 Cost Quantification 量化表格，统一指向 Tool Planning Protocol 消除跨文件重复
- **Read 小窗口提示升级** — 碎片化读取判定从 `<15%` 提高到 `<25%`，警告从温和 `Hint:` 升级为 `⚠️ FRAGMENTED READ`，新增成本警告和推荐窗口
- **RepoMap 双模式** — 新增 `/repomap` 命令（别名 `/rmap`），通用模式（默认）轻量级 Workspace Context 自动检测项目根与语言提示、随 `/cd` 动态重建；开发者模式全量 Go 符号地图持久缓存。5 维硬上限（文件/目录/超时）防御任意目录卡死。接入内联菜单补全
- **RepoMap dev 密码保护** — `/repomap dev` 需输入密码（无回显），最多 3 次提示 + 第 4 次错误直接退出防暴力破解；验证通过后 `config.json` 持久化，重启自动免密码；`/help` Workspace 区补充条目
- **上下文压缩按需激活** — `context-compression` skill 从 `always: true` 改为按需触发（archived/compressed/placeholder 等关键词），简单查询每轮节省约 15 行系统提示
- **编译检查前置** — Go File Write/Edit Pre-Check 规则：写文件前确认模块名/import/函数可见性、同轮编译、一次修完所有错误
- **验证** — `go test ./...` 通过，529 测试 0 回归

## v0.37.0（2026-06-26）

- **上下文压缩系统重构** — 新增 Large Content Archive，大型历史工具输出优先写入本地归档，payload 中替换为占位符，降低上下文占用且保留原文可恢复
- **ContextArchiveRead 工具** — 新增只读工具，可按 `arch-xxxxxx` 恢复历史归档原文，用于压缩后仍需精确代码、错误日志或工具输出的场景
- **AI 摘要胶囊** — archive 无候选时 fallback 到结构化短摘要，保留任务摘要、关键决策、文件路径、archive refs 与下一步动作，避免旧版长摘要超时和高成本
- **压缩档位控制** — `/context compression` 支持 `off/stress/low/medium/high/max/reset`，默认 `low=200K→120K`，压测可用 `stress=50K→30K`
- **Compression Backoff 防抖** — 每次有效压缩检查后 next trigger 按 50K 推进，避免低阈值下连续重复压缩，`/context cache` 可查看 steps 与 next trigger
- **上下文恢复一致性** — session snapshot 持久化 compressed blocks/watermark，`/history`、`/resume` 恢复后继续使用压缩胶囊与 archive refs，`/new` 和 clear 会同步清理相关状态
- **读取效率优化** — 最近完整 tool result 保留提升到 8 条，按工具类型设置历史压缩上限，新增 Read Range Memory 记录已读区间，减少大文件重复读取
- **工具执行优化** — 只读并行工具去重，同批重复 Read/Grep/Glob/ContextArchiveRead 只执行一次；Grep 支持 path 直接传入单个文件
- **诊断信息增强** — `/context cache` 增加 Archive、File Cache、Tool Efficiency、Compression Backoff、Last Compression 等字段，便于观察上下文占用、压缩效果和重复读取
- **AI capsule 精确细节保留** — 压缩 prompt 强化数字阈值、状态流转、函数名、文件路径、用户约束和 `/context cache` 运行态事实，降低压缩后细节混淆
- **稳定性修复** — 修复 Last Compression 空 attempt 覆盖、`/new` 继承旧压缩状态、压缩 success 后连续触发等问题
- **验证** — `go test ./... -count=1` 与 `go build ./...` 已通过，当前 529 个测试用例，0 回归
