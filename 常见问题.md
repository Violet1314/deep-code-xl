# 常见问题

---

### npm install 报错 / 下载失败

**原因**：GitHub Releases 在部分地区访问不稳定。

**解决**：

1. 重试安装：
   ```powershell
   npm install -g deep-code-xl
   ```

2. 如果多次重试仍失败，手动下载：
   - 打开 https://github.com/Violet1314/deep-code-xl/releases
   - 下载最新版本的 `deep-code.exe`
   - 放到 npm 全局目录（通常是 `%APPDATA%\npm\node_modules\deep-code-xl\`）

---

### 启动报 "deep-code.exe 未找到"

**原因**：安装时下载 `deep-code.exe` 未成功。

**解决**：按上一个问题的方法，手动下载 `deep-code.exe` 并放到对应目录。

---

### 启动后连接 API 失败

**检查清单**：

1. API Key 是否正确（不要有多余空格或换行）
2. Base URL 是否可访问（在浏览器打开试一下）
3. 网络是否需要代理

**重新配置**

```powershell
deep-code
# 进入后输入
/connect login
```

---

### 不想用某个 Provider 了，怎么换

```powershell
# 在交互界面中输入
/connect login
```

重新走配置流程即可。

---

### API 调用报错 / 模型无响应

1. 确认 API Key 额度是否用完
2. 确认模型名称是否正确
3. 尝试切换模型：`/model`

---

### 终端显示乱码

1. 确认使用 **Windows Terminal** 或新版 PowerShell（不支持旧版 cmd）
2. 确认终端编码为 UTF-8

---

### Deep Code 会收集我的数据吗

**不会。** Deep Code 完全在本地运行，API Key 和对话内容只保存在你本机的 `config.json` 和 `.deep-code/` 目录中，不会上传到任何第三方服务器。自动更新仅访问 GitHub 查询版本号。

---

### 怎么卸载

```powershell
npm uninstall -g deep-code-xl
```

配置文件需要手动删除（如需要）：

```powershell
Remove-Item -Recurse "$env:USERPROFILE\.deep-code"
```
